# CLAUDE.md — AI Scam Detector

## Project Overview

AI Scam Detector is a Web3 risk intelligence tool that analyzes smart contracts on Avalanche C-Chain and returns a risk score plus a human-readable explanation generated by AI.

The core idea: blockchain data is public but not understandable. We translate on-chain complexity into simple human decisions.

We are NOT building a block explorer, a trading tool, or a DeFi analytics platform.
We ARE building a layer that makes blockchain risk legible to non-technical users.

## Tech Stack

### Frontend
- Next.js (App Router) + TailwindCSS
- Single-page focus: input → loading → result

### Backend
- Python (FastAPI) preferred, Node/Express acceptable
- Single main endpoint: `POST /analyze`

### Blockchain — Avalanche C-Chain
- Primary: **Snowtrace API** (https://api.snowtrace.io/api) — contract verification, source code, ABI
- Holders / token data: **Covalent API** (https://api.covalenthq.com) or **Moralis**
- Direct RPC: `https://api.avax.network/ext/bc/C/rpc` (chain ID 43114)

### AI
- OpenAI GPT-4o or Claude API
- Used ONLY for explanation — never for scoring

### Storage (Nice to have)
- IPFS via Web3.Storage or nft.storage

## MVP Scope

### Must Have
- Contract address input
- Avalanche C-Chain support
- Risk score (0–100)
- Risk level: Low / Medium / High

- Wallet analysis

## Risk Scoring Engine

### Critical Rule
The LLM does NOT calculate the score. The score is computed by deterministic rules. The LLM only receives the final signals + score and explains them in plain language.

### Signals and Weights

| Signal                          | Condition                        | Points Added |
|---------------------------------|----------------------------------|-------------|
| Holder concentration            | Top 1 holder > 30%               | +20         |
| Holder concentration            | Top 10 holders > 60%             | +15         |
| Ownership                       | Owner not renounced              | +20         |
| Contract verification           | Unverified on Snowtrace          | +15         |
| Mint function                   | Present in contract              | +10         |
| Blacklist function              | Present in contract              | +10         |
| Pause function                  | Present in contract              | +5          |
| Transfer restrictions           | Detected                         | +10         |
| Liquidity pool age              | Created less than 7 days ago     | +10         |
| Liquidity amount                | Very low (< $10k)                | +10         |

Score is capped at 100.

### Risk Levels
- 0–30: Low Risk
- 31–60: Medium Risk
- 61–100: High Risk

## AI Prompt Template

```
SYSTEM:
You are a blockchain security analyst. Your job is to explain smart contract risk to non-technical users in plain, simple language. Be direct. Avoid jargon. Never invent signals that were not provided.

USER:
Analyze the following smart contract risk data and explain it clearly.

Contract address: {address}
Risk score: {score}/100
Risk level: {level}

Detected signals:
{signals_as_bullet_list}

Your response must include:
1. A 2–3 sentence plain-language explanation of why this contract is risky (or not).
2. The most important warnings, phrased so a non-technical user understands the real-world implication.
3. A one-sentence bottom line.

Do not repeat the raw signal names. Translate them into consequences the user cares about.
```

## UI Principles

- Minimal, fast, clear
- The risk score must be the first thing you see (large, color-coded)
- Color coding: green (Low), yellow/orange (Medium), red (High)
- Warnings as short bullet points
- AI explanation below warnings
- Never show raw JSON to the user

### Result Page Layout
1. Risk Score — big number, colored background
2. Risk Level — label (Low / Medium / High)
3. Warnings — bullet list
4. Explanation — AI-generated paragraph
5. (Optional) Link to IPFS report

## Demo Strategy

The demo is done by Mariano. He is the target user. That's a deliberate choice that communicates the product's value to the jury.

### Demo Script (under 3 minutes)
1. Mariano explains the problem in one sentence: "You want to buy a token, but how do you know if it's a scam?"
2. Paste a known high-risk contract → show High Risk result with warnings
3. Show the AI explanation — Mariano reads it out loud, confirms he understands it
4. Paste a safer/verified contract → show the contrast
5. Closing line: "We translate blockchain complexity into human understanding."

### Known Contracts for Demo
- Look for flagged Avalanche contracts on https://rugdoc.io or community forums
- Verify the contract shows High Risk before the demo, not during
- Have a backup contract ready in case the first one's data is unavailable
